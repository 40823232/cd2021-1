討論:

在無其他同步或非同步網際程式工具輔助下, 若採用 Ethercalc 以網際同步方式可取得各分組的學員學號, 但因無自動定組序的方法 (可類比至後續若要設法導入自動化流程讀取各組的分組網站, 則必須設法讓電腦程式可以讀取各分組的分組網站內容), 因此需要人工介入訂定組序並收集各組分組網站的連結並發布在網際環境中, 目前所面臨的問題是, 除了各組兩名組員可自選外, 其他流程 (定各班組序, 取得各班分組網站連結並自動嵌入預定的 content.htm 檔案中) 都希望以自動化方式完成.

目前已知 ethercalc-python 可協助讀取特定的 Ethercalc 表單, 以下列程式為例, 因為利用 Python 的 set 去除各列中重複的 None 值, 但因 set 資料的特性是 unordered, 因此雖達成去除重複資料的目的, 但卻弄亂原先的資料順序, 因此或可採直接讀取 Ethercalc 表單資料後, 依照各行固定位置的學員學號 (第二與第三行資料), 並設法忽略中間的空白列 (row) 內容後, 再與最終的各班選課名單 (透過 https://github.com/mdecourse/nfulist 讀取資料) 進行比對後 (因其中可能包含未自選組員的情況, 則必須由電腦進行自動配對), 再進行各班組序的訂定, 並依各組組序與組長 Github 帳號 (如何取得?), 自動產生各組的分組倉儲連結與分組網站連結後, 設法產生對應的 html 資料後, 再設法嵌入指定的 content.htm 中.

分組協同

當 stage1 兩人一組進行協同時, 希望各學員 Github 帳號下都帶有分組倉儲與分組網站, 則可以透過 git remote add, 設法讓各組員 Github 帳號下的分組網站內容同步.

說明針對之前使用舊版 cmsimde 的學員進行更新的操作步驟:

除了利用 git remote add 設法從分組網站近端倉儲版本建立內容同步各學員分組網站外, 也可以利用 fork 及反向 pull request 讓學員附屬分組網站與主分組網站的內容保持同步.

Stage1

實習目的:

  1. 讓學員利用電腦輔助機械設計套件執行協同任務, 找出保存各設計資料版本的有效方法
  2. 讓學員利用最有效率的方式展示協同產品設計過程的各相關細節與關鍵技術
  3. 讓學員實習如何協同編寫淺顯易懂且內容充實的產品設計簡報與結案報告 (Reveal.js 與 latex-paper 研究)
  4. 讓學員在協同設計過程, 利用 ssh 維護個人與分組倉儲與網站
  5. 讓學員利用 git remote add 建立 Github 與 Gitlab 上的同步倉儲與協同設計網站

能不能夠更方便?

上述修改 CMSiMDE 啟動, 版本提交推送與目錄內容複製的動機, 就是希望電腦執行能夠更人性化, 也就是能讓各項操作更具效率, 先前使用者必須:

    cd cmsimde

    python wsgi.py

將這兩個動作合併, 以:

    python cmsimde/wsgi.py

批次檔案 cms.bat 執行, 使用者只要輸入 cms 然後按下 enter 就可以將兩個動作減為止輸入三個單字, 當然使用者若再將 cms.bat 改為 c.bat 也是可以:-)

而 acp.bat 執行時隨後必須輸入提交訊息字串的動作除了批次檔案的名稱可以再簡化為 a.bat 之外, 似乎已經省無可省, 但是使用者在開啟動態系統後, 通常若在近端則需要以瀏覽器開啟 https://localhost:9443, 因此若能在開啟動態網站的同時, 在批次檔案中同時開啟瀏覽器, 又可以省下不少時間, 在批次檔案中同時執行命令的機制稱為 pipe.

假如採用 pipe 除了啟動 wsgi.py 外也利用 Firefox 瀏覽器開啟 https://localhost:9443, 則可以將 cms.bat 改為:

python cmsimde/wsgi.py 1> nul | "C:\Program Files (x86)\Mozilla Firefox\firefox" https://localhost:9443

SSH 了沒

先前在 http://mde.tw/cp2020/content/SSH.html 曾經說明過如何使用 SSH push 近端的改版資料到 Github, 但進入協同產品設計實習課程之後, 仍然有許多人還是沒能採用 SSH 對 Github 連線. 可見之前的說明仍然不夠清楚明白.

在這裡我們看看能不能找到更方便的方法來使用 SSH 對 Github 連線. 也就是直接利用 putty.exe 與 puttygen.exe 連線來產生對應的認證檔案.

以下為操作步驟:

  1. 請登入 Github 並且選擇帳號右下方 Settings - SSH and GPG keys 備用
  2. 在 start_ipv6.bat 加入 set GIT_SSH=%Disk%:\putty\plink.exe
  3. 開啟 Putty 目錄中的 puttygen.exe, 以滑鼠點擊 Generate 後, 讓滑鼠在上方空白處隨意移動, 即可建立 key, 利用滑鼠選擇 key 區域中的文字資料, 並複製到 SciTE 編輯區, 存為  id_rsa 檔, 並將此內容貼至 Github SSH key 欄位中後儲存.
  4. 接下來在上一步驟中 puttygen.exe 視窗下方, 選擇 Save private key 後, 將 key 存為 my_key.ppk 或"學號.ppk"備用.
  5. 接下來則需要在 putty.exe 中建立一個 session, 用於以 ssh 協定連線到 github.com, 而 session 名稱可以自行命名, 但是必須與倉儲 .git 目錄中 config 的連線 url session 名稱對應.
  6. 當利用 putty.exe 設定連線 session 時必須注意是否使用 Proxy, 以及 SSH 中的 Auth 是否對應到前面所存檔的 .ppk private key 位置.

以上設定的重要關鍵為:

  1. 啟動隨身系統時, 是否已經設定 GIT_SSH 環境變數, 指名要透過 plink.exe 以 putty 作為 ssh client.
  2. Github 接受 OpenSSH 格式的 key, 而 putty 則接受 .ppk 格式的 key.

接著來看幾個專有名詞:

SSH - Secure SHell 這是安全的"殼"嗎? 是的, 穿過安全的殼層, 就可以直通核心! 那麼有安全的殼層, 也就是有"不安全"的殼層嗎? 先說到底甚麼是殼層 (Shell):

從 https://en.wikipedia.org/wiki/Shell_(computing) 直接 copy 第一行解釋:

In computing, a shell is a computer program which exposes an operating system's services to a human user or other program.

哇, 原來 Shell 是一個電腦程式, 可以讓使用者直接使用操作系統的服務.

假如針對 Shell 的說明頁, 再往內 copy 兩段說明, 真相就可以大白了:

In general, operating system shells use either a command-line interface (CLI) or graphical user interface (GUI), depending on a computer's role and particular operation. It is named a shell because it is the outermost layer around the operating system.

用中文說明:

  1. 之所以稱為"殼層", 是因為 Shell 程式位於操作系統的最外層, 進入這一層就可以操縱操作系統的各項服務.
  2. 操作系統的殼層程式可以透過 CLI (指令介面) 或 GUI (圖形介面) 執行.

假如 SSH 是 Secure SHell, 那麼不安全的"殼"層又是什麼? 是 Telnet! 現在應該已經沒有人用 telnet 對遠端操作系統 (Linux, BSD 或其他 Unix 類操作系統), 而是使用 ssh.

接著讓大家練習一下使用 ssh 對 Ubuntu 操作系統連線, 使用其 Service.

假如您們小組想要自行安裝 Virtualbox 上的 Ubuntu 20.04 伺服器版本, 可以從 http://c.kmol.info/downloads/  下載 ubuntu-20.04.2-live-server-amd64.iso 後試著安裝設定. (需要摸索一下, 這裡有參考書)

或者直接從 http://c.kmol.info/downloads/ 下載 ub2021_vdi.7z 解開後就是已經安裝完成的 Ubuntu 20.04 操作系統 vdi 檔案.

上述 Virtualbox 上的 Ubuntu 除了可以用於練習 ssh 對 Ubuntu 連線外, 後續也將用於 Fossil SCM server 與 Windows 10 client 的實習.

hello_ethercalc.py

#!/usr/bin/env python3
import ethercalc
import pprint

pp = pprint.PrettyPrinter(indent=4)
e = ethercalc.EtherCalc("http://140.130.17.17:8000")
# http://140.130.17.17:8000/kkcao0lc2ew7
print(e)
# 設定 id 為 kkcao0lc2ew7 的 sheet C1 值為 number 2
e.command("kkcao0lc2ew7", ["set C1 value n 2"])
# C2 設為字串 "電腦輔助設計", 但是字串前面必須加一個單引號
e.command("kkcao0lc2ew7", [ethercalc.set("C2", "'電腦輔助設計")])
e.command("kkcao0lc2ew7", [ethercalc.set("C3", 3),
                                           ethercalc.set("C4", 4),
                                           # C5 為 C3+C4
                                           ethercalc.set("C5", "=C3+C4"),
                                          ])
pp.pprint(e.cells("kkcao0lc2ew7"))
pp.pprint(e.cells("kkcao0lc2ew7", "C5"))
pp.pprint(e.export("kkcao0lc2ew7"))

有關 ethercalc:

若 ethercalc 與 redis 合用, sheet 資料將會存入 /var/lib/redis/dump.rdb, 此一檔案儲存設定位於 /etc/redis/redis.conf 中的 dbfilename dump.rdb

使用者若要以 root 身分檢視 /var/lib/redis/dump.rdb 可以透過 sudo -s 以 root 身分執行命令.

若以 /etc/init.d/redis-server stop 關閉 redis, 然後執行 ethercalc, sheet 資料將會存在執行命令目錄下的 dump 目錄中.

在結合 redis 使用下的 ethercalc, 若希望 reset /var/lib/redis/dump.rdb 中的資料:

sudo -s

/etc/init.d/redis-server stop

rm /var/lib/redis/dump.rdb

/etc/init.d/redis-server start

ethercalc

則 /var/lib/redis/dump.rdb 中為空資料.

import os
findText = """content to remove"""
for dname, dirs, files in os.walk("path_name"):
    for fname in files:
        if fname[-5:] == ".html":
            print(fname)
            fpath = os.path.join(dname, fname)
            with open(fpath, encoding="utf-8") as f:
                s = f.read()
            s = s.replace(findText, "")
            with open(fpath, "w", encoding="utf-8") as f:
                f.write(s)

# rename_according_file_time.py
import os
from pathlib import Path

# put this file outside 2021-03_2a_stage1 directory
files = []
# directory name
dname = "2021-03_2a_stage1"
# iterdir() iterate over the files in directory
paths = sorted(Path(dname).iterdir(), key=os.path.getmtime)
for i in paths:
    #os.fspath() eturn the file system representation of the path
    files.append(os.fspath(i))
#print(files)
i = 0
for fpath in files:
    # original file name is fname
    i += 1
    # new file name
    npath = os.path.join(dname, "cd2021_stage1_2a_g" + str(i) + ".mp4")
    #print(npath)
    os.rename(fpath, npath)

# get_video_duration.py
# pip install moviepy
import os
from moviepy.editor import VideoFileClip

# Converts into more readable format
def convert(seconds):
    hours = seconds // 3600
    seconds %= 3600
    mins = seconds // 60
    seconds %= 60
    return hours, mins, seconds
    
for dname, dirs, files in os.walk("2021-03_2a_stage1"):
    for fname in files:
        vName = os.path.join(dname, fname)
        clip = VideoFileClip(vName)
        hours, mins, secs = convert(clip.duration)
        print(fname + "組:", str(int(mins)) + "分", str(int(secs)) + "秒")
